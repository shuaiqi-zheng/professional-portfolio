Artifact 6 — Pre-Trained Model Decision Matrix

This visual artifact compares pre-trained models across NLP, computer vision, and tabular AI to support practical model selection based on size, accuracy, speed, and explainability.

Artifact Focus: The best pre-trained model is not always the largest or most accurate model. Selection should match the deployment constraint, domain, latency requirement, and explainability need.

Project Overview

This artifact was created for AIML-501 Model Development. It extends the explainability portfolio work by comparing model-selection trade-offs across NLP, computer vision, and tabular data. The matrix shows how size, benchmark accuracy, speed, and interpretability affect practical deployment decisions.

Artifact Metadata

Objective

Compare representative pre-trained models and identify which model fits different deployment scenarios.

Process

Models were grouped by domain, then evaluated using published size, benchmark performance, speed indicators, and explainability fit.

Tools Used

Research synthesis, decision-matrix design, infographic development, HTML, CSS, and GitHub Pages.

Value Proposition

The matrix helps teams avoid choosing a model from benchmark scores alone. It connects technical model characteristics to practitioner decisions, such as using DistilBERT for fast NLP classification, MobileNetV2 for real-time vision, EfficientNetB0 for balanced vision transfer learning, and LightGBM or XGBoost for structured tabular scoring.

Key Takeaway

Model selection is a trade-off decision. Larger models can improve accuracy, but smaller or more interpretable models may be better when latency, cost, deployment environment, or auditability matter more.
